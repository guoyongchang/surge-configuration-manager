# 设置菜单拆分与增强

## 背景
当前设置页面混杂了 HTTP 监听、MITM、HOST、URL REWRITE、云同步等功能，用户根本不知道这些功能的存在。将其拆分为独立菜单可提升可发现性和可用性。

## 范围
### 包含
- 侧边栏独立菜单：HTTP 监听、MITM、HOST、URL REWRITE、云同步
- HOST 功能：
  - 单个新增（弹窗）
  - 批量新增（弹窗）
  - 批量编辑（弹窗）
  - 多选删除
  - 禁用/启用
  - 云同步完整支持
- URL REWRITE 功能：
  - 单个新增（弹窗）
  - 批量新增（弹窗）
  - 批量编辑（弹窗）
  - 多选删除
  - 禁用/启用
  - 云同步完整支持
- HTTP 监听：独立菜单页面
- MITM：独立菜单页面
- 云同步：独立菜单页面

### 不包含
- 无

## 验收标准
- [ ] AC-01: 侧边栏显示独立的 HTTP 监听、MITM、HOST、URL REWRITE、云同步菜单
- [ ] AC-02: HOST 支持单个新增、批量新增、批量编辑、多选删除、禁用/启用
- [ ] AC-03: URL REWRITE 支持单个新增、批量新增、批量编辑、多选删除、禁用/启用
- [ ] AC-04: HOST 和 URL REWRITE 数据支持完整云同步
- [ ] AC-05: `pnpm test` passes

## 技术备注
- 参考现有 Rules 页面和 ExtraNodes 页面的实现模式
- HOST 和 URL REWRITE 数据结构需要设计
- 云同步逻辑复用现有 cloud-sync 模块
