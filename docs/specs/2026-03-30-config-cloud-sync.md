# 配置云同步

## 背景

用户需要在多台 Mac 设备间同步 Surge 配置，包括订阅源（含内容）、规则集、额外节点和最终产物。换电脑或设备损坏时可通过云端恢复。

## 范围

### 包含

- 手动同步：用户点击按钮触发同步
- 自动同步：生成配置后自动触发（可关闭）
- 冲突检测与解决：显示本地/云端两版本，用户二选一
- GitHub 私有仓库存储：PAT 认证，仓库级 Scope Token
- 多文件目录结构：
  - `subscriptions/` — 订阅源配置（含解析后的真实内容）
  - `rules/` — 远程规则集 URL 列表 + 个别路由规则
  - `nodes/` — 额外节点配置
  - `output/` — 最终生成的 `.conf` 文件
- 从云端恢复：输入仓库地址，拉取并覆盖本地数据
- 同步状态指示：显示上次同步时间、冲突标记

### 不包含

- GitHub OAuth 登录（仅 PAT）
- 自动创建仓库（用户手动创建空仓库）
- 按字段/规则级别的冲突合并（仅整版本二选一）
- 多账号管理（同一时刻仅一个仓库配置）

## 验收标准

- [ ] AC-01: 用户在设置页面填入 GitHub PAT 和仓库地址后，可成功连接并显示同步状态
- [ ] AC-02: 手动点击"同步"按钮，将本地数据推送到 GitHub 仓库
- [ ] AC-03: 生成配置后自动同步（如设置开启）
- [ ] AC-04: 从云端拉取数据并覆盖本地（恢复功能）
- [ ] AC-05: 检测到云端与本地版本不一致时，显示两版本供用户选择
- [ ] AC-06: 用户确认冲突解决方案后，完成同步并更新时间戳
- [ ] AC-07: `pnpm test` passes

## 技术备注

- GitHub API 使用 `PUT /repos/{owner}/{repo}/contents/{path}` 原子性更新文件
- 冲突检测：比较本地 `last_modified` 与云端文件 SHA，不一致则触发冲突流程
- PAT 权限：`repo` scope 可读写指定仓库
- 同步锁：同步进行中禁止再次触发
