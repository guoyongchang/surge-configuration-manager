# 配置文件版本回退

## 背景
用户在 Output 页面生成合并配置后，如果误操作覆盖了 Surge 配置文件，希望能够回退到之前的版本，或比对两个版本之间的差异。

## 范围
### 包含
- **自动备份**：生成 .conf 文件时，若与上次备份内容不一致，则自动保存一份到备份目录
- **历史版本列表**：在 Output 页面展示所有历史备份（时间戳、文件名）
- **版本预览**：点击历史版本可预览其内容
- **一键回退**：选择历史版本后，一键恢复到 Surge 配置目录

### 不包含
- 自动定时备份
- 分支/合并功能
- 远程推送
- 版本差异比对功能

## 验收标准
- [ ] AC-01：Output 页面显示「历史版本」按钮或列表入口
- [ ] AC-02：点击历史版本可预览该版本的文件内容
- [ ] AC-03：选择历史版本后可一键回退到 Surge 配置目录
- [ ] AC-04：生成 .conf 时，若内容与上次备份不一致，自动创建新备份
- [ ] AC-05：`pnpm test` passes

## 技术备注
- 备份目录建议：`{app_data}/backups/`，按时间戳命名（如 `2026-03-30_143052.conf`）
- 备份文件命名格式：`{日期}_{时间}.conf`
- 判断「不一致」：通过文件内容 hash 或完整内容比对
- 回退操作：读取选中备份文件内容，写入 Surge 配置目录（覆盖或替换）
